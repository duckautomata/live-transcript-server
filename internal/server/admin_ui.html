What each part of this page shows

Worker
The transcription worker assigned to this channel. Online means it has reported its status within the last 5 minutes; the line below shows its version and when it last checked in. Offline usually means the worker process is down or can't reach the server.
Discord bot
The server-side bot that watches Discord for stream announcements and queues them automatically. Online (green) means its connection to Discord is healthy and this channel's announcements will be picked up. Reconnecting (amber) means Discord has gone quiet and the server is re-establishing the connection. Offline (red) means the connection is down — the reason is shown below the status, and announcements are missed until it recovers, so queue URLs by hand in the meantime. Off (gray) means no bot is configured on this server at all. An amber dot next to Online flags a configuration problem (for example, this channel has no announcement name mapped) even though the bot itself is healthy.
Connected clients
How many viewers currently hold a live WebSocket connection to this channel's transcript page. Clients connecting or disconnecting don't trigger an instant update — this count refreshes about once a minute.
Server
The version and build time of the transcript server itself (this backend, not the worker).
Streams
Every stream stored for this channel. Live means the worker is transcribing it right now; Past means it has ended. Each row shows the title, the stream ID, when it started, and what kind of media was captured.
Incoming queue
Stream URLs waiting for the worker. The worker takes a URL out of the queue when it picks it up; Remove takes one out by hand before that happens. The form at the bottom queues a new URL.
Worker controls
The Restart and Stop signals for the worker, plus a badge showing whether a restart is pending. See the “Restart vs. Stop” card below for what each one does.
Membership keys (tab)
Only shown when this channel has membership configured and the archive server is reachable. Manages the keys that gate members-only content — see the “Membership keys” card below.

Queue, stream, and restart changes show up within a second or two — no need to reload. The connected-client count, the worker's last-seen time, and the Discord bot status are the exception: they refresh about once a minute, plus whenever you perform an action or return to this tab.

Start a stream (queue it)

  1. Paste the full stream URL (e.g. https://twitch.tv/… or https://www.youtube.com/…) into the input at the bottom of the Incoming queue card and click Add.
  2. The worker is notified within seconds. As soon as it isn't busy with another stream, it takes the URL from the queue and starts capturing.
  3. The stream then appears in Streams with a Live pill once transcription begins.

Adding a URL that's already queued just re-queues it — no duplicates. If the worker is offline, the URL simply waits in the queue until the worker comes back.

Delete a stream

Past (inactive) stream:

  1. Click Delete on its row in the Streams card.
  2. In the confirmation, choose whether to also delete its media files (kept by default — see the next card).
  3. Confirm. The stream and its transcript are removed from the server, and connected viewers drop it immediately.

Live (active) stream: the Delete button is disabled while a stream is live — the worker would just re-create it on its next push. Instead:

  1. Click Stop current stream under Worker controls and confirm.
  2. Wait for the stream's pill to flip from Live to Past — typically a few seconds; the page updates on its own.
  3. Delete it like any past stream.

Delete media from a stream

Media (the audio, video, clip, and frame files in storage) is deleted together with its stream — there is no separate media-only delete.

  1. Click Delete on the stream's row.
  2. Tick “Also delete media files from storage” in the confirmation dialog.
  3. Confirm — the stream's entire storage folder is permanently removed.

Left unticked (the default), only the stream record and transcript are deleted; every media file stays in storage.

Worker controls: Restart vs. Stop

Restart worker
Sets a flag the worker reads on its next poll, making it drop whatever it's doing and recycle. The incoming queue is untouched, so anything queued gets picked right back up. Use this to un-stick a misbehaving worker.
Stop current stream
Clears every URL from the incoming queue and sets the restart flag. The worker drops the current stream and then sits idle, since nothing is left to pick up. Use this to end a capture on purpose — and as step one of deleting a live stream.
Restart pending / Cancel
While a restart is waiting for the worker, the card header shows Restart pending with a Cancel button. Cancel withdraws the request if the worker hasn't acted on it yet; the badge clears on its own once the worker restarts.

Membership keys

The Membership keys tab appears only when this channel has membership configured and the archive server is reachable. Keys gate access to this channel's members-only content on the archive server.

Generate new key
Creates a key on the archive server and copies it to your clipboard. The archive keeps at most 2 keys per channel, so generating a third silently removes the oldest. To rotate: generate a new key, hand it out, and let the old one get pruned by the rotation after that.
Copy
Copies that row's key to the clipboard. Each row also shows when the key expires.
Delete all keys
Removes every key for the channel at once — there is no per-key delete. Anyone using one of the keys loses access immediately.

Login & the admin key

Everything on this page is protected by this channel's admin key, sent along with every request. The pill in the top-right corner shows whether your key is currently accepted.

“Save key in this browser” keeps the key in this browser's local storage so you skip the login next time on this device — leave it unchecked on shared computers. Logout forgets the saved key and returns to the login prompt.
